HTML Fundamentals

Basic HTML Structure

The basic structure of an HTML document includes the doctype declaration, the root <html> element, and the <head> and <body> elements.

<!DOCTYPE html>
Declares the document type and version of HTML being used. 
<!DOCTYPE html>
<html>
Root element of an HTML page, containing all other elements. 
<html lang="en">
    <!-- content -->
  </html>
<head>
Contains meta information about the HTML document, such as the title and character set. 
<head>
    <meta charset="UTF-8">
    <title>My Page</title>
  </head>
<body>
Contains the visible content of the HTML document. 
<body>
    <h1>My Page</h1>
    <p>This is a paragraph.</p>
  </body>

Head Elements

Elements that contain meta information about the HTML document.

<title>
Specifies the title of the HTML document, which appears in the browser tab. 
<title>My Website</title>
<meta>
Provides meta information about the HTML document. 
<meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1">
<link>
Defines the relationship between the current document and an external resource, such as a stylesheet. 
<link rel="stylesheet" href="styles.css">
<script>
References an external JavaScript file or contains inline JavaScript code. 
<script src="script.js"></script>
<style>
Contains internal CSS styles for the HTML document. 
<style>
    body {
      background-color: #f0f0f0;
    }
  </style>
<base>
Specifies the base URL to use for all relative URLs in the document. 
<base href="https://example.com/" target="_blank">
<noscript>
Provides alternative content for users who have disabled JavaScript in the browser or whose browsers do not support JavaScript. 
<noscript>
    <p>Your browser does not support JavaScript.</p>
  </noscript>

Block-level Text Elements

Elements that represent blocks of content in the document.

<h1> to <h6>
Heading tags from largest to smallest. 
<h1>Main Heading</h1>
  <h2>Subheading</h2>
  <h3>Section Title</h3>
  <h4>Sub-section</h4>
  <h5>Detail Level</h5>
  <h6>Smallest Heading</h6>
<p>
Defines a paragraph of text. Browsers automatically add spacing before and after it. 
<p>This is a paragraph of text.</p>
<pre>
Displays preformatted text. Preserves spaces, tabs, and line breaks exactly as written in the HTML source. 
<pre>
  Code block 
  with indentation.
  Write actual line breaks in your HTML.
  </pre>
<blockquote>
Represents a long quotation from another source. Often displayed with indentation. The optional cite attribute indicates the source URL. 
<blockquote cite="https://example.com">
    <p>This is a quote.</p>
  </blockquote>
<address>
Provides contact information for the author or owner of the document. 
<address>
    Email: name@example.com
  </address>
<hr>
Creates a thematic break (horizontal rule) between paragraph-level elements. 
<h2>Introduction</h2>
  <p>This is the intro text.</p>

  <hr>

  <h2>Details</h2>
  <p>More information here.</p>
<div>
A generic block-level container used to group elements for styling or layout purposes. Has no semantic meaning by itself. 
<div class="container">
    <!-- content -->
  </div>

Inline Text Elements

Inline formatting and semantic text elements.

<span>
A generic inline container with no semantic meaning. Used mainly for styling or grouping text with CSS/JS. 
<p>This is <span class="highlight">highlighted</span> text.</p>
<strong>
Indicates strong importance or seriousness. Screen readers emphasise it, and browsers render it bold by default. 
<strong>Important text</strong>
<b>
Bold text for visual styling only, without semantic emphasis. Use <strong> when meaning matters. 
<b>Bold text.</b>
<em>
Marks text with stress or emphasis. Screen readers often change tone, and browsers render it italic by default. 
<em>Emphasized</em>
<i>
Italic text for stylistic offset (foreign words, technical terms, names). 
<i>Italic text</i>
<mark>
Highlights text as relevant or noteworthy. Default style is yellow background. 
<mark>Highlighted text</mark>
<small>
Represents side comments, disclaimers, or fine print. Typically rendered in smaller font size. 
<small>© 2026 Example Company</small>
<code>
Inline code snippet. Displays text in a monospace font to indicate programming or markup. 
<code>console.log("Hello, World!")</code>
<abbr>
Abbreviation or acronym. The title attribute provides the full form for accessibility. 
<abbr title="HyperText Markup Language">HTML</abbr>
<cite>
Citation or reference to a creative work (book, article, film, etc.). 
<cite>Citation examples</cite>
<q>
Short inline quotation. Browsers usually add quotation marks automatically. 
<q>This is a quote</q>
<time>
Represents a specific time or date. The datetime attribute makes it machine-readable. 
<time datetime="2025-01-01">January 1, 2025</time>
<br>
Inserts a line break inside text. It forces the next part of the content to start on a new line without creating a new paragraph. 
Line 1<br>Line 2

Lists

HTML lists are used to group a set of related items (unordered, ordered, and description lists).

<ul>
An unordered list. Contains a set of list items without a specific order. 
<ul>
    <li>Item 1</li>
    <li>Item 2</li>
  </ul>
<ol>
Creates an ordered list, usually numbered. 
<ol>
    <li>First item</li>
    <li>Second item</li>
  </ol>
<li>
Represents a single list item. Must be placed inside <ul>, <ol>, or <menu>. 
<li>List item.</li>
<dl>
Defines a description list, used for pairs of terms and their definitions. 
<dl>
    <dt>Term</dt>
    <dd>Definition</dd>
  </dl>
<dt>
Represents a term or name in a description list. 
<dt>Term</dt>
<dd>
Provides the definition, explanation, or value associated with a <dt> term. 
<dd>Definition of the term</dd>

Links

Hyperlinks and anchor elements

<a>
Defines a hyperlink. Used to link to pages, files, email addresses, or sections within the same page. The href attribute specifies the destination, and target controls how the link opens. 
<a href="https://example.com" target="_blank">Visit Website</a>
  <a href="#section-id">Jump to Section</a>

Media Elements

Elements for embedding audio, video, and other media content

<img>
Embeds an image in the document. The alt attribute provides alternative text for accessibility. Use the src attribute to specify the image URL and width and height attributes to control the display size. 
<img src="image.jpg" alt="Description of the image">
  <img src="photo.jpg" alt="Photo description" width="500" height="300">
<picture>
Container for providing multiple responsive image options. 
<picture>
    <source media="(max-width: 800px)" srcset="large.jpg">
    <source media="(min-width: 400px)" srcset="medium.jpg">
    <img src="small.jpg" alt="Responsive image">
  </picture>
<figure>
Represents a self-contained content element, often used with an image and a caption. 
<figure>
    <img src="image.jpg" alt="Description">
    <figcaption>Caption text</figcaption>
  </figure>
<figcaption>
Provides a caption or description for a <figure> element. 
<figcaption>Image caption text</figcaption>
<video>
Embeds a video player. The controls attribute displays playback controls. Use multiple <source> elements for responsive or cross-format support. 
<video controls width="100%">
    <source src="video.mp4" type="video/mp4">
    Your browser does not support the video tag.
  </video>
<audio>
Embeds an audio player. Supports multiple formats for compatibility. 
<audio controls>
    <source src="audio.mp3" type="audio/mpeg">
    Your browser does not support the audio element.
  </audio>
<track>
Adds text tracks to video or audio, such as subtitles, captions, or descriptions. 
<track src="subtitles_en.vtt" kind="subtitles" srclang="en" label="English">
<svg>
Container for scalable vector graphics. Ideal for icons, shapes, and illustrations that remain sharp at any size. 
<svg width="100" height="100">
    <circle cx="50" cy="50" r="40" fill="blue"/>
  </svg>
<source>
Defines alternative media sources for <video>, <audio>, or <picture>. The browser selects the most appropriate file. 
<source src="video.mp4" type="video/mp4">
<map>
Defines an image map with clickable areas. Must be associated with an image using the usemap attribute. 
<map name="example-map">
    <area shape="rect" coords="0,0,100,100" href="page.html" alt="Clickable area">
  </map>
<area>
Defines a clickable region inside an image map. Coordinates depend on the shape used. 
<area shape="circle" coords="75, 75, 50" href="link.html" alt="Clickable area">

Embedded Content

Embedding external pages, media, and interactive objects

<iframe>
Embeds another webpage or interactive content inside the current page. Commonly used for maps, videos, widgets, and external tools. Always include a descriptive title for accessibility. 
<iframe
    src="https://example.com"
    width="300"
    height="200"
    title="Embedded content">
  </iframe>
<embed>
Embeds external content such as PDFs or browser plugins. Mostly outdated and replaced by modern HTML5 elements. 
<embed src="document.pdf" type="application/pdf" width="400" height="300">
<object>
Embeds external resources such as PDFs, images, or multimedia. More flexible than <embed>, but less commonly used today. 
<object data="document.pdf" type="application/pdf" width="400" height="300">
    <p>Fallback content if the file cannot be displayed.</p>
  </object>
<param>
Defines parameters for <object> elements. Deprecated in modern HTML and rarely used. 
<param name="autoplay" value="true">

Tables

Creating structured data displays

<table>
Creates a structured data display with rows and columns. Useful for presenting tabular information in a clear and organized manner. 
<table>
    <thead>
      <tr>
        <th>Name</th>
        <th>Age</th>
      </tr>
    </thead>
    <tbody>
      <tr>
        <td>John</td>
        <td>25</td>
      </tr>
    </tbody>
  </table>
<caption>
Provides a title or description for a table. Improves accessibility and helps users understand the table's purpose. 
<table>
    <caption>Employee Information</caption>
    <thead>
      <tr>
        <th>Name</th>
        <th>Age</th>
      </tr>
    </thead>
    <tbody>
      <tr>
        <td>John</td>
        <td>25</td>
      </tr>
    </tbody>
  </table>
<thead>
Groups the header content in a table. Improves readability and semantic structure. 
<thead>
    <tr>
      <th>Name</th>
      <th>Age</th>
    </tr>
  </thead>
<tbody>
Groups the body content in a table. Improves readability and semantic structure. 
<tbody>
    <tr>
      <td>John</td>
      <td>25</td>
    </tr>
  </tbody>
<tr>
Represents a single row in a table. Contains <th> or <td> elements. 
<tr>
    <td>John</td>
    <td>25</td>
  </tr>
<th>
Defines a header cell in a table. Typically displayed in bold and centered. Use scope for accessibility. 
<th scope="col">Header</th>
<td>
Represents a standard data cell in a table. Contains the actual data for the row. 
<td>John</td>
<tfoot>
Groups the footer content in a table. Useful for displaying summary information or additional details. 
<tfoot>
    <tr>
      <td>Total</td>
      <td>25</td>
    </tr>
  </tfoot>
<colgroup>
Groups columns in a table. Useful for applying styles or attributes to multiple columns at once. 
<colgroup>
    <col span="2">
    <col span="2">
  </colgroup>
<col>
Defines a single column within a table. Can be used with <colgroup> to apply styles or attributes to multiple columns. 
<col style="background-color: lightgray;">

Forms

Creating interactive forms for collecting user input

<form>
Defines an HTML form for collecting user input. Contains various form controls such as text fields, checkboxes, and buttons. The action attribute specifies where to send the form data, and the method attribute defines how to send it (GET or POST). 
<form action="/submit" method="post">
    <label for="name">Name:</label>
    <input type="text" id="name" name="name">
    <button type="submit">Submit</button>
  </form>
<input>
Defines an input field for collecting user data. Can be of various types such as text, email, password, number, etc. 
<input type="text" name="username">
  <input type="email" name="email" required>
  <input type="password" name="password">
  <input type="number" name="age" min="0" max="100">
  <input type="checkbox" name="agree">
  <input type="file" name="upload" accept="image/*">
<label>
Associates text with a form control. Improves accessibility and usability. 
<label for="username">Username:</label>
  <input type="text" id="username" name="username">
<textarea>
Defines a multi-line text input field. Useful for collecting larger amounts of text. 
<textarea name="message" rows="5" cols="40">
    Enter your message here...
  </textarea>
<select>
Creates a drop-down list for selecting one or more options. 
<select name="country">
    <option value="us">United States</option>
    <option value="ca">Canada</option>
    <option value="uk">United Kingdom</option>
  </select>
<button>
Creates a clickable button.
<button type="submit">Send</button>
<option>
Defines an item inside a <select> drop-down list. 
<option value="1">One</option>
<optgroup>
Groups related options inside a <select> drop-down list. 
<optgroup label="Fruits">
    <option value="apple">Apple</option>
    <option value="banana">Banana</option>
  </optgroup>
<datalist>
Provides a list of predefined options for an <input> element (autocomplete suggestions). 
<input list="browsers" name="browser">
  <datalist id="browsers">
    <option value="Chrome">
    <option value="Firefox">
    <option value="Safari">
  </datalist>
<fieldset>
Groups related form elements together. 
<fieldset>
    <legend>Fruits</legend>
    <input type="radio" id="apple" name="fruit" value="apple">
    <label for="apple">Apple</label>
    <input type="radio" id="banana" name="fruit" value="banana">
    <label for="banana">Banana</label>
  </fieldset>
<legend>
Provides a label for a <fieldset>. 
<legend>Fruits</legend>
<output>
Displays the result of a calculation or the value of a form field. 
<output name="result" for="a b">0</output>
<progress>
Represents the progress of a task. 
<progress value="70" max="100"></progress>
<meter>
Represents a scalar measurement within a known range. 
<meter value="70" min="0" max="100"></meter>

Semantic Elements

HTML5 elements that provide meaning and structure to web content

<header>
Represents the header of a section or page. Typically contains a heading and introductory content. 
<header>
    <h1>My Website</h1>
    <p>Welcome to my website!</p>
  </header>
<nav>
Defines a set of navigation links. Should be used for primary or secondary menus. 
<nav>
    <ul>
      <li><a href="#home">Home</a></li>
      <li><a href="#about">About</a></li>
      <li><a href="#contact">Contact</a></li>
    </ul>
  </nav>
<main>
Represents the main content of a document. Only one <main> element should be used per page. 
<main>
    <h1>My Article</h1>
    <p>This is the main content of the article.</p>
  </main>
<section>
Represents a thematic grouping of content, typically with a heading. 
<section>
    <h2>Section Title</h2>
  </section>
<article>
Defines self-contained, independent content such as blog posts, news articles, or forum entries. 
<article>
    <h2>Blog Post</h2>
  </article>
<aside>
Contains content that is tangentially related to the main content (sidebars, advertisements...). 
<aside>
    Sidebar content
  </aside>
<footer>
Represents the footer of a page or section. Often contains copyright, contact info, or links. 
<footer>
    <p>© 2025</p>
    <address>Contact info</address>
  </footer>
<details>
Creates a collapsible disclosure widget. 
<details>
    <summary>Click to expand</summary>
    Hidden content
  </details>
<summary>
Provides the visible heading for a <details> element. 
<summary>More Information</summary>
<dialog>
Represents a dialog box or window. 
<dialog open>
    Modal content goes here.
  </dialog>
<search>
Represents a search section. Still experimental and not widely supported. 
<search>
    <input type="search">
  </search>

Elements used for scripts, templates, and web components

<script>
Embeds client-side scripts. Can contain JavaScript inline or reference an external file with the src attribute. 
<script>
    // Your JavaScript code here
  </script>
<noscript>
Provides fallback content for users who have disabled JavaScript. 
<noscript>
    <p>JavaScript is disabled in your browser.</p>
  </noscript>
<template>
Defines reusable HTML content that can be cloned later. 
<template>
    <div class="card">
      <h3></h3>
      <p></p>
    </div>
  </template>
<slot>
Placeholder for content in Web Components. 
<slot name="header"></slot>

Common Global Attributes

Attributes that can be applied to almost any HTML element

id
Unique identifier for an element. Used for linking, scripting, and styling. 
<div id="main">Content</div>
class
Assigns one or more class names for styling or JavaScript selection. 
<div class="container">Content</div>
style
Applies inline CSS styles directly to an element. 
<div style="color: blue; padding: 10px;">Styled text</div>
hidden
Hides the element from the user interface while keeping it in the DOM. 
<div hidden>This content is hidden.</div>
contenteditable
Makes an element editable by the user. 
<div contenteditable="true">Click to edit</div>
data-*
Custom data attributes for storing private data on the element. 
<div data-id="123" data-user="john">Content</div>
draggable
Allows the element to be dragged with the mouse. 
<div draggable="true">Drag me</div>
dir
Sets the text direction (ltr, rtl, or auto). 
<div dir="rtl">Texte de droite à gauche</div>
tabindex
Controls the order of keyboard navigation (focus). 
<div tabindex="0">Focusable element</div>
accesskey
Specifies a keyboard shortcut to activate or focus the element. 
<button accesskey="s">Save</button>
spellcheck
Enables or disables spell checking for editable content. 
<textarea spellcheck="true"></textarea>
translate
Indicates whether the element's content should be translated. 
<div translate="no">Technical term</div>

CSS Fundamentals

Selectors

Methods to target HTML elements using CSS.

Element Selector
Targets elements by their tag name. 
/* Selects all <h1> elements */
  h1 {
    color: blue;
  }
Class Selector
Selects elements with a specific class. 
/* Selects all elements with the class "btn" */
  .btn {
    font-size: 16px;
  }

  /* Multiple classes */
  .class1.class2 {
    color: red;
  }
ID Selector
Selects an element with a specific ID (unique on the page). 
#header {
    background: yellow;
  }
Universal Selector
Targets all elements on the page. 
* {
    margin: 0;
    padding: 0;
    box-sizing: border-box;
  }
Descendant Selector
Selects elements nested inside another element (any level). 
div p {
    color: red;
  }
Child Selector
Selects only direct children.
p > span {
    font-weight: bold;
  }
Adjacent Sibling Selector
Selects the element immediately following another. 
h1 + p {
    margin-top: 0;
  }
General Sibling Selector
Selects all siblings that follow another element. 
h1 ~ p {
    color: gray;
  }
Attribute Selectors
Selects elements based on the presence or value of attributes. 
input[type="text"] {
    border: 1px solid gray;
  }

  /* Common patterns */
  a[href^="https"] { color: green; }     /* starts with */
  a[href$=".pdf"]   { color: red; }      /* ends with */
  img[alt*="logo"]  { border: 2px solid gold; } /* contains */
  [target="_blank"] { color: purple; }
Pseudo-classes
Selects elements based on state or structural position. 
/* Interactive states */
  a:hover, a:focus { color: red; }
  input:focus-visible { outline: 3px solid blue; }

  /* Structural */
  li:first-child { font-weight: bold; }
  li:last-child { margin-bottom: 0; }
  li:nth-child(odd) { background: #f9f9f9; }

  /* Other useful states */
  input:disabled { opacity: 0.5; }
  :focus-within { background: #f0f8ff; }

  /* Logical */
  :not(.special) { color: gray; }
  :is(header, main, footer) h2 { font-size: 1.8rem; }
Pseudo-elements
Style specific parts of an element. 
p::before {
    content: "→ ";
    color: blue;
  }

  p::after {
    content: " ✓";
  }

  p::first-line { font-weight: bold; }
  p::first-letter { font-size: 2.5em; }
  ::selection { background: yellow; color: black; }
Group Selector
Select multiple selectors at once. 
h1, h2, h3, .title {
    color: blue;
  }
:is() / :where() / :has()
Modern logical selectors.
:is(header, main, footer) h2 {
    font-size: 2rem;
  }

  :where(.card, .panel) :hover {
    transform: scale(1.02);
  }

Colors & Backgrounds

Color and background properties.

color
Sets the text color of an element. 
p {
    color: red;
    color: tomato;
    color: #333;
    color: #ff5733;
    color: rgb(255 87 51);
    color: rgba(255 87 51 / 0.8);
    color: hsl(9 100% 60%);
    color: hsla(9 100% 60% / 0.8);
    color: oklch(70% 0.2 240); /* Modern color space */
  }
background-color
Sets the background color of an element. 
background-color: #f0f0f0;
  background-color: rgb(240 240 240);
  background-color: hsl(0 0% 94%);
background-image
Sets the background image of an element. 
background-image: url("image.jpg");
  background-image: url("overlay.png"), url("bg.jpg"); /* multiple */

  /* Gradients */
  background-image: linear-gradient(to right, red, blue);
  background-image: radial-gradient(circle at center, red, blue);
  background-image: conic-gradient(red, yellow, green);
background-size / position / repeat
Controls how background images behave. 
background-size: cover | contain | 100px 200px | 50% auto;
  background-position: center | top right | 50% 50%;
  background-repeat: no-repeat | repeat-x | repeat-y;
background-clip / origin
Defines painting and positioning areas. 
background-clip: border-box | padding-box | content-box | text;
  background-origin: border-box | padding-box | content-box;
Gradiens (advanced)
Creates smooth transitions between colors. Can be linear, radial, conic, or repeating. Defines color stops, angles, shapes, and positions for complex backgrounds without images. 
linear-gradient(45deg, red, blue);
  radial-gradient(circle, red 0%, blue 100%);
  conic-gradient(from 90deg, red, yellow, green);
  repeating-linear-gradient(45deg, #333 0px, #333 10px, #666 10px, #666 20px);
background
Shorthand that combines all background properties into one declaration. Supports color, images, gradients, position, size, repeat, attachment, origin, clip, and multiple layers. 
/* Basic shorthand */
  background: #fff url("img.jpg") no-repeat center / cover;

  /* Gradient only */
  background: linear-gradient(135deg, #667eea, #764ba2);

  /* Multiple layers (top → bottom) */
  background:
    url("texture.png") repeat,
    linear-gradient(to bottom, #0006, #0000),
    #222;

  /* Full syntax example */
  background:
    #f0f0f0
    url("photo.jpg")
    center / contain
    no-repeat
    fixed
    padding-box
    content-box;

  /* Values you can use */
  background-color: #fff | transparent | rgb() | hsl();
  background-image: url() | linear-gradient() | radial-gradient() | none;
  background-position: left | center | 20px 40px | 50% 50%;
  background-size: cover | contain | 100px 200px | auto;
  background-repeat: repeat | no-repeat | repeat-x | repeat-y | space | round;
  background-attachment: scroll | fixed | local;
  background-origin: border-box | padding-box | content-box;
  background-clip: border-box | padding-box | content-box;

Text & Fonts

Properties for styling text and fonts.

font-family
Specifies the font family for an element. 
body {
    font-family: system-ui, -apple-system, "Segoe UI", Roboto, sans-serif;
    font-family: "Roboto", "Helvetica Neue", Arial, sans-serif;
    font-family: "Open Sans", sans-serif;
  }
font-size
Controls text size (supports responsive units). 
font-size: 16px;
  font-size: 1rem;
  font-size: 2.5em;
  font-size: clamp(1rem, 2.5vw, 2rem);
font-weight / font-style
Controls thickness and slant of the font. 
font-weight: bold | normal | 100 | 200 | 300 | 400 | 500 | 600 | 700 | 800 | 900;
  font-style: italic | oblique | normal;
font
Shorthand for setting font properties. 
body {
    font: italic small-caps bold 16px/1.5 "Open Sans", sans-serif;
  }
text-align / text-decoration
Controls text alignment and decoration. 
text-align: left | center | right | justify;
  text-decoration: underline wavy red 2px;
  text-decoration: none;
text-shadow / line-height
Controls text shadow and line height. 
text-shadow: 2px 2px 4px rgba(0 0 0 / 0.5);
  line-height: 1.6 | 1.8 | 2;
@font-face
Allows you to define custom fonts by specifying a font family name and a source URL. Useful for using web fonts that are not installed on the user's system. 
@font-face {
    font-family: "MyCustomFont";
    src: url("my-custom-font.woff2") format("woff2"),
        url("my-custom-font.woff") format("woff");
    font-weight: 400;
    font-style: normal;
    font-display: swap;
  }

Box Model

Spacing, sizing, and layout fundamentals of CSS

width / height
Controls the size of the content box. 
width: 300px;
  width: 50%;
  width: 100vw;
  width: auto;

  width: fit-content;
  width: min-content;
  width: max-content;

  min-width: 200px;
  max-width: 1200px;

  height: 200px;
  height: 100vh;
  min-height: 400px;
  max-height: 80vh;
margin
Controls external spacing. Vertical margins collapse between adjacent block elements. 
margin: 20px;                    /* all sides */
  margin: 10px 20px;                /* top/bottom | left/right */
  margin: 10px 20px 30px;           /* top | left/right | bottom */
  margin: 10px 20px 30px 40px;      /* top | right | bottom | left */

  margin: 0 auto;                   /* horizontal centering */
  margin-inline: auto;              /* logical property (modern) */
padding
Controls internal spacing (between content and border). 
padding: 20px;                    /* all sides */
  padding: 10px 20px;                /* top/bottom | left/right */
  padding: 10px 20px 30px;           /* top | left/right | bottom */
  padding: 10px 20px 30px 40px;      /* top | right | bottom | left */
border
Defines the border around an element (width, style, color). 
border: 2px solid #333;
  border: 3px dashed red;

  border-width: 2px;
  border-style: solid | dashed | dotted | double | none;
  border-color: #333;

  border-radius: 8px;
  border-radius: 50%;                    /* perfect circle */
  border-radius: 10px 30px 20px 40px;    /* individual corners */
box-sizing
Controls how width/height are calculated. 
box-sizing: content-box; /* default (padding + border added) */
  box-sizing: border-box;   /* includes padding + border (recommended) */
box-shadow
Adds shadow effects around an element. 
box-shadow: 5px 5px 10px rgba(0 0 0 / 0.3);
  box-shadow: 0 0 15px rgba(0 0 0 / 0.2);
  box-shadow: inset 0 0 10px rgba(0 0 0 / 0.5); /* inner shadow */
outline
Creates an outline outside the border (does not affect layout). 
outline: 2px solid #333;
  outline: 3px dashed blue;
  outline: none; /* often used to remove default focus outline */
overflow
Controls how overflowing content is handled. 
overflow: visible; /* default */
  overflow: hidden;
  overflow: scroll;
  overflow: auto;
Logical Properties
Modern properties that adapt to writing direction (LTR/RTL). 
margin-block: 1rem 2rem;   /* top | bottom */
  margin-inline: 1.5rem;     /* left | right */

  padding-block: 20px;
  border-block: 2px solid black;
  border-inline-start: 4px solid blue;
aspect-ratio
Defines the aspect ratio of an element. 
aspect-ratio: 16 / 9;
  aspect-ratio: 1 / 1; /* square */
  aspect-ratio: auto;

Display & Visibility

Display modes and visibility

display
Defines how an element is displayed. 
display: block;
  display: inline;
  display: inline-block;
  display: none;
  display: flex;
  display: grid;
  display: contents;
  display: flow-root;
visibility
Controls visibility without removing the element from layout. 
visibility: visible;
  visibility: hidden;   /* keeps space */
  visibility: collapse; /* for tables */
opacity
Controls transparency (0 = invisible, 1 = fully visible). 
opacity: 1;     /* default */
  opacity: 0.8;
  opacity: 0;     /* invisible but still takes space */

Positioning

Element positioning strategies

position
Specifies the positioning method. 
position: static;     /* default */
  position: relative;
  position: absolute;
  position: fixed;
  position: sticky;
top / right / bottom / left / inset
Offset values for positioned elements. 
position: absolute;
  top: 0;
  right: 0;
  bottom: 20px;
  left: 20px;

  /* Modern shorthand */
  inset: 10px 20px 30px 40px; /* top right bottom left */
z-index
Controls stacking order of positioned elements. 
z-index: -1;
  z-index: 0;
  z-index: 10;
  z-index: 9999;
float & clear
Legacy layout technique (largely replaced by Flexbox/Grid). 
.float-left {
    float: left;
    margin-right: 15px;
  }

  .clearfix::after {
    content: "";
    display: table;
    clear: both;
  }

Overflow

Content overflow behavior

overflow
Controls how content that exceeds its container is handled. 
div {
    overflow: visible;     /* default - content spills out */
    overflow: hidden;      /* clips content */
    overflow: scroll;      /* always shows scrollbars */
    overflow: auto;        /* shows scrollbars only when needed */
    
    /* Independent axes */
    overflow-x: hidden;
    overflow-y: auto;
    
    /* Modern */
    overflow: clip;        /* clips without allowing scroll */
  }
overflow-wrap
Controls how long words or strings are wrapped. 
.text {
    overflow-wrap: normal;      /* default */
    overflow-wrap: break-word;  /* breaks long words if needed */
    overflow-wrap: anywhere;    /* more aggressive breaking */
    
    word-break: break-all;      /* breaks at any character */
    word-break: keep-all;       /* preserves words (CJK) */
  }
text-overflow
Truncates overflowing text with an ellipsis. 
.ellipsis {
    white-space: nowrap;      /* important */
    overflow: hidden;
    text-overflow: ellipsis;
  }

  /* Multi-line truncation */
  .line-clamp {
    display: -webkit-box;
    -webkit-line-clamp: 3;
    -webkit-box-orient: vertical;
    overflow: hidden;
  }

Flexbox

One-dimensional layout model

Container Properties
Properties applied to the flex container. 
.flex-container {
    display: flex;                    /* or inline-flex */
    flex-direction: row | column | row-reverse | column-reverse;
    flex-wrap: nowrap | wrap | wrap-reverse;
    flex-flow: row wrap;              /* shorthand */
    
    justify-content: flex-start | center | flex-end | space-between | space-around | space-evenly;
    align-items: stretch | flex-start | center | flex-end | baseline;
    align-content: flex-start | center | space-between;
    
    gap: 1.5rem;                      /* modern */
    row-gap: 1rem;
    column-gap: 2rem;
  }
Item Properties
Properties applied to flex items. 
.flex-item {
    flex: 1 1 auto;                   /* grow | shrink | basis */
    flex-grow: 1;
    flex-shrink: 0;
    flex-basis: 200px;
    
    align-self: stretch | center | flex-start | flex-end;
    order: 0;                         /* change visual order */
  }
Common Flex Patterns
Frequently used Flexbox patterns. 
.center {
    display: flex;
    justify-content: center;
    align-items: center;
  }

  .space-between {
    display: flex;
    justify-content: space-between;
    align-items: center;
  }

CSS Grid

Two-dimensional layout model

Grid Container Properties
Properties applied to the grid container. 
.grid-container {
    display: grid;
    grid-template-columns: 1fr 2fr 1fr;
    grid-template-columns: repeat(3, 1fr);
    grid-template-columns: repeat(auto-fit, minmax(250px, 1fr));
    
    grid-template-rows: 100px auto 150px;
    
    gap: 1.5rem;
    row-gap: 20px;
    column-gap: 30px;
    
    justify-items: center;
    align-items: start;
    
    grid-auto-flow: row | column | dense;
  }
Grid Item Properties
Properties applied to grid items. 
.grid-item {
    grid-column: 1 / 3;             /* start / end */
    grid-column: span 2;            /* span 2 columns */
    
    grid-row: 1 / -1;               /* span all rows */
    
    grid-area: header;              /* named area */
    
    justify-self: center;
    align-self: end;
    place-self: center;             /* shorthand */
  }
grid-template-areas
Defines named areas for easier layout. 
.page {
    display: grid;
    grid-template-areas:
      "header header"
      "sidebar main"
      "footer footer";
    grid-template-columns: 250px 1fr;
  }

Transforms

2D and 3D visual transformations

transform
Applies 2D or 3D transformations (does not affect layout). 
/* 2D */
  transform: translate(50px, 100px);
  transform: rotate(45deg);
  transform: scale(1.5);
  transform: skew(10deg, 5deg);

  /* Multiple transforms */
  transform: translate(50px, 100px) rotate(45deg) scale(1.2);

  /* 3D */
  transform: rotateX(45deg);
  transform: rotateY(45deg);
  transform: translateZ(100px);
  transform: rotate3d(1, 1, 1, 45deg);
transform-origin
Defines the origin point of the transformation. 
transform: rotate(45deg);
  transform-origin: center;
  transform-origin: top left;
  transform-origin: 50% 50%;
3D Properties
Properties for creating 3D effects. 
.parent {
    perspective: 1000px;
    transform-style: preserve-3d;
  }

  .card {
    backface-visibility: hidden;
  }

Transitions

Smooth animations between CSS property values

transition
Shorthand for smooth property changes. 
button {
    transition: all 0.3s ease;
    transition: transform 0.2s ease, opacity 0.3s;
    transition: background-color 0.4s cubic-bezier(0.4, 0, 0.2, 1);
  }

  button:hover {
    transform: scale(1.05);
    background-color: #3b82f6;
  }
Transition Properties
Individual transition settings.
transition-property: transform, opacity, background-color;
  transition-duration: 0.3s;
  transition-timing-function: ease-out;
  transition-delay: 100ms;
Timing Functions
Controls the speed curve of the transition. 
ease            /* default - smooth */
  ease-in         /* slow start */
  ease-out        /* slow end */
  ease-in-out     /* smooth acceleration */
  linear          /* constant speed */
  cubic-bezier(0.4, 0, 0.2, 1); /* custom */

Animations

Complex, multi-step animations using keyframes

animation
Shorthand to apply animations.
.element {
    animation: slideIn 0.6s ease-out forwards;
    animation: bounce 2s infinite;
    animation: spin 1.5s linear infinite;
  }
@keyframes
Defines the animation sequence.
@keyframes slideIn {
    from {
      opacity: 0;
      transform: translateX(-100%);
    }
    to {
      opacity: 1;
      transform: translateX(0);
    }
  }

  @keyframes spin {
    to {
      transform: rotate(360deg);
    }
  }
Animation Properties
Detailed control over animations. 
animation-name: slideIn;
  animation-duration: 0.6s;
  animation-timing-function: ease-out;
  animation-delay: 0.2s;
  animation-iteration-count: infinite;
  animation-fill-mode: forwards;

Responsive Design

Techniques for creating layouts that adapt to different screen sizes.

@media
Conditional styling based on viewport size, device features, or user preferences. 
@media (min-width: 768px) {
    .layout { grid-template-columns: 1fr 2fr; }
  }

  @media (prefers-color-scheme: dark) {
    body { background: #111; color: #fff; }
  }

  @media (prefers-reduced-motion: reduce) {
    * { animation: none !important; }
  }
@container
Container queries — responsive styles based on parent container size. 
.card-container {
    container-type: inline-size;
    container-name: card;
  }

  @container card (min-width: 400px) {
    .card { display: flex; }
  }
Viewport Units
Fluid units that scale with the viewport. 
section {
    padding: 5vw;
  }

  .hero {
    height: 100dvh;     /* dynamic viewport height */
  }
clamp()
Fluid scaling between min and max values. 
h1 {
    font-size: clamp(1.8rem, 4vw, 3.5rem);
  }

  .container {
    width: clamp(300px, 90%, 1400px);
  }
auto-fit / auto-fill
Responsive grids without media queries. 
.grid {
    display: grid;
    gap: 1rem;
    grid-template-columns: repeat(auto-fit, minmax(250px, 1fr));
  }
Responsive Images
Best practices for images that adapt to screen size. 
img {
    max-width: 100%;
    height: auto;
    object-fit: cover;
  }
Logical Properties
Direction-aware properties (better for RTL languages). 
.box {
    padding-inline: 1rem;     /* left/right */
    margin-block: 2rem;       /* top/bottom */
    border-inline: 2px solid #333;
  }
